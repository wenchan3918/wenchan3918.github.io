發表於2023-04-04|更新於2023-04-04

Python3.6.8使用各種系統編譯的映像檔比較

系統 Ubuntu 22.04
(build python source code)		 Stretch Slim Stretch Alpine 3.10 Alpine 3.10 with Multi-Stage Build
映像大小 996M 1006MB 380MB 336MB 222MB
  • Ubuntu 22.04 LTS（Jammy Jellyfish）基于Debian 12（bookworm）。
  • Ubuntu 21.04（Hirsute Hippo）基于Debian 11（bullseye）。
  • Ubuntu 20.04 LTS（Focal Fossa）基于Debian 10（Buster）。
  • Ubuntu 18.04 LTS（Bionic Beaver）基于Debian 9（Stretch）。
  • Ubuntu 16.04 LTS（Xenial Xerus）基于Debian 8（Jessie）。
  • slim: Slim鏡像是基於官方鏡像（如Debian或Alpine）構建的鏡像，但已經去除了大多數不必要的組件和文件。這意味著我們的鏡像將比官方鏡像小得多，並且只包含我們應用程序所需的內容。
  • buster:Debian 10, does not support python3.6
  • stretch:Debian 9
  • jessie:Debian 8
  • Alpine: Alpine Linux非常適合作為基礎鏡像，因為它非常輕量級。它只包含最基本的軟件包和工具，而其他軟件包則可以根據需要進行安裝。此外，Alpine Linux還專注於安全，因此它的軟件包經過了精心的篩选和審查，以確保它們是安全和穩定的。
  • Docker多階段構建（Multi-Stage Build）是一種構建Docker鏡像的技術，它可以使得構建出的鏡像更加精簡、安全和高效。通過使用多階段構建，我們可以在單個Dockerfile中定義多個構建階段，每個階段都有自己的環境和上下文。
